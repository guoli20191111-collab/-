# 03 · Step2 上传照片

> 页面文件：`step2.html`
> 页面路径：Step1b 选择角色 → **Step2 上传照片** → Step3 AI 生成等待
> 设计原则：操作聚焦，上传即走

---

## 页面功能说明

| 功能项 | 说明 |
|--------|------|
| 顶部导航栏 | Logo + IP 剧集下拉 + 头像（未登录：默认头像 + 「未登录」文字，点击弹出登录弹窗；已登录：用户照片） |
| 标题行 | 「上传照片 · 生成古风形象」 + 返回按钮 |
| 上传区域 | 虚线框，支持点击上传或拖拽上传，限 JPG / PNG |
| 上传预览 | 上传成功后显示缩略图 + 「✓ 上传成功」 + 「重新上传」链接 |
| 「下一步」按钮 | 未上传时置灰，上传后激活，点击跳转生成页 |
| 背景视频轮播 | 3 段古风视频循环（与首页共用） |
| 底部进度条 | 进度 66% |

### 交互逻辑

1. 进入页面 → 显示上传占位区
2. 点击上传区域 → 打开文件选择器（JPG/PNG）
3. 或拖拽文件到上传区域 → 自动识别
4. 上传成功 → 图片压缩至 1200px，转 base64 存入 `localStorage`
5. 显示预览图 + 「重新上传」
6. 点击「重新上传」→ 清空，回到上传占位状态
7. 点击「下一步」→ 跳转 `step3.html`
8. 点击「返回」→ 回到 `step1b.html`

### 数据存储

| Key | 值 | 说明 |
|-----|---|------|
| `userPhoto` | `data:image/jpeg;base64,...` | 压缩后的用户照片 |
